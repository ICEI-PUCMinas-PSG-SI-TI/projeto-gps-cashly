# Documentação do Projeto

## Visão Geral do Projeto

Cashly é o projeto de desenvolvimento de uma solução (aplicação) para gestão de pagamentos/fluxo financeiro destinada ao cliente interno do curso/trabalho. O repositório contém o código-fonte em `src/` e a documentação técnica e de gestão no diretório `docs/`.

Objetivos principais:
- Entregar uma aplicação funcional para visualização e gestão de transações financeiras (MVP).
- Documentar o escopo, planejamento, execução, monitoramento e encerramento do projeto.
- Garantir qualidade, testes e aceite do cliente/professor.

Escopo resumido:
- Desenvolvimento das funcionalidades principais no frontend/backend presentes em `src/`.
- Testes de integração e aceitação.
- Entregáveis de documentação (TAP, termo de encerramento, solicitações de compra e escopo) integrados em `docs/`.

Stakeholders:
- Equipe de desenvolvimento (repositório `src/`).
- Orientador/Professor (cliente do trabalho acadêmico).
- Patrocinador/Departamento responsável.
